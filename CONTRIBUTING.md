# Contributing

Thanks for your interest in haal-installer.

## Honest expectations

This is a side project maintained in spare time. Issues and pull requests are welcome, but I cannot guarantee they will be reviewed or merged in any reasonable timeframe. If you need a fix or feature urgently, **forking is the right move** — the license allows it and I actively encourage it.

## The best way to contribute

Fork the repo and build what you need. If your changes could benefit others, open a PR — it may get picked up eventually.

## If you open an issue

- Check existing issues first
- Be specific: what did you expect, what happened, what OS and tool versions
- No timeline guarantees

## If you open a PR

- Keep it focused — one thing per PR
- Make sure it builds (`cargo tauri build` or `npm run build`)
- A short description of why, not just what

## Registry contributions

If you want to share skills, rules, or commands with the community, the better path is to **publish your own registry** and list it in your seed's `repos-manifest.json`. You don't need to contribute to this repo to share content — that's the whole point of the multi-registry model. See [docs/multi-registry.md](docs/multi-registry.md).
